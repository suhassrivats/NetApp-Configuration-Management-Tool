Startup and Memory Management APIs for Python
The Startup API is used to create and initialize the object that is required to communicate between the NetApp Manageability SDK client and the NetApp storage system. This page contains information about the Startup Management APIs for Python, including the description and syntax, and the input, input types, input description, and input return values.
NaServer
Prototype
NaServer(server, majorversion, minorversion)
Description
Creates a new object of NaServer class and sets the default value for the following object members:
user = root
password = ""
style = LOGIN
port = 80
transport_type = HTTP
servertype= FILER
You can overwrite the default values by using the set Core APIs.
Input
Description
server
The name or the IP address of the server in IPv4 or IPv6 format
majorversion
The major version of the ONTAP API
For OnCommand Unified Manager server, use 1 as the major version.
minorversion
The minor version of the ONTAP API
For OnCommand Unified Manager server, use 0 as the minor version.
Return value
The NaServer object
