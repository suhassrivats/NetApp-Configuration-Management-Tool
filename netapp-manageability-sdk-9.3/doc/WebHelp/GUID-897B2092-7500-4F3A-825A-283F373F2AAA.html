Status Request APIs for Python
Status Request APIs for Python are used to retrieve the status of an API requested to the server, and to get the error number and a string containing the error details in case there is a request failure. This page contains information about the Status Request Management APIs for Python, including the description and syntax, and the input, input types, input description, and input return values.
NaElement.results_status
Prototype
results_status()
Description
Indicates success or failure of an API call, such as invoke and invoke_elem.
Return Value
◦
passed
◦
failed
NaElement::results_errno
Prototype
results_errno()
Description
Returns an error number of an API call, such as invoke or invoke_elem.
Return Value
•
0 if the API call is successful
•
The error number, if the API call fails
NaElement.results_reason
Prototype
results_reason()
Description
Gets the reason for failure in a string.
The string is present if results_status does not return passed.
Return Value
A string describing the error
